## Why GitHub Pages might not be working — fix it here

### Step 1 — Make sure Pages is enabled correctly
1. Go to your repo on GitHub
2. Click **Settings** (top menu)
3. Scroll down to **Pages** in the left sidebar
4. Under **Source**, make sure it says:
   - Branch: `main` (not `master` or anything else)
   - Folder: `/ (root)`
5. Click **Save** if you changed anything

### Step 2 — Make sure the file is named exactly `index.html`
- The file **must** be named `index.html` (all lowercase)
- If it's named `Index.html` or `index.HTML` it won't work

### Step 3 — Make sure it's in the root of the repo
- `index.html` should be at the **top level** of the repo, not inside a folder
- Correct: `your-repo/index.html`
- Wrong:   `your-repo/files/index.html`

### Step 4 — Wait 1-2 minutes and hard refresh
- GitHub Pages takes up to 2 minutes to publish after each update
- After waiting, press Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac) to bypass browser cache

### Step 5 — Check the Actions tab for errors
- In your repo, click the **Actions** tab
- If you see a red X next to a recent deploy, click it to see the error

### Step 6 — Confirm your site URL
Your URL will be:
  https://YOUR-GITHUB-USERNAME.github.io/YOUR-REPO-NAME/

Both username and repo name are case-sensitive.
